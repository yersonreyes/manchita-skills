# Roadmap de Prototipado

## ¿Qué es?

El Roadmap de Prototipado es un plan visual que muestra la secuencia yTimeline de los prototipos a crear, desde la idea hasta el producto final. Define qué se prototipa, cuándo y con qué nivel de fidelidad.

## Cómo se usa

### Estructura:

```
ROADMAP DE PROTOTIPADO
├── Fase 1: Exploración (Semana 1-2)
│   ├── Prototipo A: Sketch en papel
│   └── Prototipo B: Wireframe basic
├── Fase 2: Validación (Semana 3-4)
│   ├── Prototipo C: Clickable mid-fi
│   └── Prototipo D: User testing
├── Fase 3: Refinamiento (Semana 5-6)
│   ├── Prototipo E: High-fi mockup
│   └── Prototipo F: Prototipo funcional
└── Fase 4: Lanzamiento (Semana 7-8)
    └── MVP: Producto mínimo viable
```

### Criterios para definir el roadmap:

- **Prioridad de features** — Cuál es más crítico?
- **Riesgo** — Qué tiene mayor uncertainty?
- **Recursos** — Cuánto tiempo y personas?
- **Dependencias** — Qué debe terminarse antes?

### Usos:

- Planificar el trabajo de diseño
- ComunicarTimeline al equipo
- Identificar bottlenecks
- Alinear expectativas con stakeholders
