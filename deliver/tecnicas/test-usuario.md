# Test de Usuario

## ¿Qué es?

El Test de Usuario es una técnica de validación donde usuarios reales interactúan con el diseño (prototipo o producto) mientras el equipo observa, documenta y analiza su comportamiento, reacciones y feedback.

## Cómo se usa

### Tipos de test:

| Tipo | Descripción | Cuándo usarlo |
|------|-------------|---------------|
| Moderado | Facilitador guía al usuario | Exploratorio |
| No moderado | Usuario hace solo | Escalable |
| Remoto | Online, anywhere | Budget limitado |
| Presencial | En persona, laboratorio | Observación detallada |

### Formatos:

- **Lab** — En espacio controlado
- **Field** — En contexto real
- **Remote** — Online via herramienta
- **Guerilla** — En espacios públicos

### Proceso:

```
1. DEFINIR TAREA      → Qué va a hacer el usuario
2. RECLUTAR          → 5-10 usuarios target
3. PREPARAR         → Prototipo, guía, setup
4. EJECUTAR         → Observar y documentar
5. ANALIZAR         → Identificar patrones
6. ITERAR           → Mejorar el diseño
```

### Métricas comunes:

- **Tasa de éxito** — Completó la tarea?
- **Tiempo** — Cuánto tardó?
- **Errores** — Cuántos cometió?
- **Satisfacción** — Cómo se sintió?

###黄金法则 del test:

> "No ayudes al usuario. Solo observa."

Deja que el usuario cometa errores para encontrar problemas reales.
