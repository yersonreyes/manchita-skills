# Diagrama de In/Out

## ¿Qué es?

El Diagrama de In/Out es una herramienta simple pero poderosa para definir los límites y flujos de un sistema, producto o proceso. Muestra qué entra (inputs) y qué sale (outputs).

## Cómo se usa

### Estructura básica:

```
    INPUTS              PROCESO               OUTPUTS
    ┌───────┐          ┌─────────┐          ┌───────┐
    │       │ ───────→ │         │ ──────→ │       │
    │ datos │  ──────→ │  SISTEMA │ ──────→ │result │
    │       │ ───────→ │  (esto   │ ──────→ │       │
    └───────┘          │  diseñas)│          └───────┘
                      └─────────┘
```

### Tipos de Inputs:

- **Información** — Datos, feedback, requisitos
- **Recursos** — Dinero, tiempo, personas
- **Materiales** — Contenido, assets

### Tipos de Outputs:

- **Producto/Servicio** — Lo que entregas
- **Datos** — Métricas, analytics
- **Feedback** — Insights, learning

## Ejemplo: App de Delivery

| Inputs | Proceso | Outputs |
|--------|---------|---------|
| Pedido del usuario | Cooking | Entrega al cliente |
| Ubicación GPS | Routing | Tiempo estimado |
| Método de pago | Payment | Confirmación |
| Inventory | Fulfillment | Notificación |

## Cuándo usarlo

- En **Discover** para entender el flujo actual
- Para definir scope de un producto
- En análisis de procesos
- Para identificar gaps en el sistema
