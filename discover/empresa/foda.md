# FODA (SWOT)

## 📖 ¿Qué es?

El **FODA** (también conocido como **SWOT** por sus siglas en inglés) es una herramienta de análisis estratégico desarrollada en la década de 1960 en la Stanford Research Institute. Se utiliza para evaluar los factores internos (fortalezas y debilidades) y externos (oportunidades y amenazas) de una organización, proyecto, producto o incluso una idea de negocio.

Esta herramienta es fundamental en el pensamiento estratégico porque permite obtener una visión panorámica de la situación actual y generar estrategias basadas en la intersección de estos cuatro elementos. Su popularidad se debe a su simplicidad y efectividad: permite resumir en una sola matriz una gran cantidad de información compleja.

El FODA es especialmente valioso en procesos de Design Thinking porque ayuda a contextualizar el problema dentro del entorno competitivo y las capacidades reales del equipo u organización.

## 🔧 Cómo se usa

### Paso a paso:

1. **Reunir al equipo** — Mínimo 3-5 personas con diferentes perspectivas (marketing, producto, ingeniería, negocio)
2. **Definir el alcance** — ¿Analizamos toda la empresa, un producto específico, o un proyecto?
3. **Analizar el entorno interno** — Identificar fortalezas y debilidades
4. **Analizar el entorno externo** — Identificar oportunidades y amenazas
5. **Cruzar los análisis** — Crear estrategias basadas en las combinaciones
6. **Priorizar** — Enfocarse en las estrategias más impactantes y factibles

### Cómo llenar cada cuadrante:

**Fortalezas (interno, positivo):**
- ¿Qué hacemos mejor que otros?
- ¿Qué recursos únicos tenemos?
- ¿Qué capacidades nos diferencian?

**Debilidades (interno, negativo):**
- ¿Qué nos falta?
- ¿Qué hacemos mal?
- ¿Qué recursos necesitamos?

**Oportunidades (externo, positivo):**
- ¿Qué tendencias podemos aprovechar?
- ¿Qué cambios en el mercado nos benefician?
- ¿Qué espacios de mercado están sin atender?

**Amenazas (externo, negativo):**
- ¿Qué competencia nos limita?
- ¿Qué cambios tecnológicos nos afectan?
- ¿Qué regulación nos impacta?

### Consejos:

- **Sé honesto** — El FODA solo funciona si es realista
- **Involucra al equipo** — Múltiples perspectivas = mejor análisis
- **Cuantifica cuando sea posible** — Números > opiniones
- **Revisa periódicamente** — El entorno cambia, el FODA también debe cambiar

## 💡 Ejemplo de uso

### Contexto:
Una startup de fintech quiere lanzar una aplicación de pagos para freelancers en Latinoamérica.

### Matriz FODA resultante:

```
                        ENTORNO
                    ┌─────────────┬─────────────┐
                    │  FORTALEZAS │  DEBILIDADES│
       INTERNO       │             │             │
                    │ • Equipo con│ • Falta de   │
                    │   experiencia│   capital    │
                    │   en pagos  │ • Marca      │
                    │ • Tecnología│   nueva       │
                    │   moderna   │ • Equipo     │
                    │ • Modelo de │   pequeño    │
                    │   pricing   │             │
                    │   flexible  │             │
                    ├─────────────┼─────────────┤
                    │ OPORTUNIDADES│  AMENAZAS   │
                    │             │             │
                    │ • Mercado   │ • Competencia│
                    │   huge en   │   establecida│
                    │   LATAM     │ • Regulación │
                    │ • Pocos     │   compleja   │
                    │   players   │ • Volatilidad│
                    │ • тенденция │   económica  │
                    │   digital   │             │
                    └─────────────┴─────────────┘
```

### Estrategias cruzada derivadas:

| | Oportunidades | Amenazas |
|---|---|---|
| **Fortalezas** | **Estrategia F-O (Ataque):** Usar tecnología moderna y experiencia para capturar mercado antes que competidores establecidos | **Estrategia F-T (Defensa):** Usar flexibilidad de pricing para competir contra players establecidos |
| **Decilidades** | **Estrategia D-O (Reforzar):** Buscar inversión para capitalizar la oportunidad de mercado | **Estrategia D-T (Supervivencia):** Enfocarse en un nicho específico para evitar confrontación directa |

### Decisión estratégica:
> Priorizar el segmento de freelancers independientes en un solo país primero (estrategia D-O) para validar el modelo antes de escalar.

## 📅 Cuándo usarlo

### Fases recomendadas:

- **Discover** — Para entender el contexto competitivo y estratégico
- **Define** — Para informar el challenge statement
- **Develop** — Para evaluar viabilidad de soluciones
- **Deliver** — Para validar que la solución es sostenible

### Situaciones ideales:

- ✅ Al iniciar un nuevo proyecto o producto
- ✅ Para evaluaciones periódicas de negocio (trimestral/semestral)
- ✅ Antes de lanzar una nueva funcionalidad significativa
- ✅ Al entrar a un nuevo mercado
- ✅ En workshops de estrategia de equipo
- ✅ Durante procesos de pivotaje
- ✅ Para comunicar la situación a stakeholders
