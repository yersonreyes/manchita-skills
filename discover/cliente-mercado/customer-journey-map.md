# Customer Journey Map (Mapa de Viaje del Cliente)

## 📖 ¿Qué es?

El **Customer Journey Map** (CJM) es una representación visual de la experiencia de un usuario a lo largo del tiempo, desde que descubre un producto/servicio hasta que lo usa y más allá. Es una de las herramientas más poderosas del Design Thinking porque pone al usuario en el centro del análisis.

Este mapa muestra las **etapas** del viaje, las **acciones** que realiza el usuario, las **emociones** que experimenta, los **touchpoints** (puntos de contacto), los **pain points** (puntos de dolor), y las **oportunidades** de mejora. La diferencia clave con otras herramientas es que el CJM captura la experiencia **temporal** y **emocional**, no solo los hechos.

El origen del Customer Journey Map se remonta a la década de 1990 con el trabajo de Michael H. L. L. T. en el campo de Service Design, y se popularizó con las metodologías de diseño de experiencia de usuario (UX) en los 2000s.

## 🔧 Cómo se usa

### Paso a paso:

1. **Define el escenario** — ¿Qué journey vas a mapear? (Ej: "Nuevo usuario intentando comprar en la app")
2. **Elige el usuario** — Usa una Persona específica, no un usuario genérico
3. **Investiga** — Obtén datos reales: entrevistas, encuestas, analytics, soporte
4. **Identifica las etapas** — Las fases del viaje (típicamente 4-7 etapas)
5. **Mapea cada etapa** — Acciones, emociones, touchpoints, pain points
6. **Encuentra oportunidades** — ¿Qué se puede mejorar?
7. **Valida con usuarios reales** — Verifica que tu mapa refleja la realidad

### Componentes del CJM:

| Componente | Descripción | Ejemplo |
|------------|-------------|---------|
| **Etapas** | Fases del viaje | Awareness → Consideración → Decisión → Uso → Lealtad |
| **Acciones** | Qué hace el usuario | Busca información, compara opciones, completa el checkout |
| **Emociones** | Cómo se siente | Confuso, emocionado, frustrado, satisfecho |
| **Touchpoints** | Puntos de contacto | App, web, email, chat, tienda física |
| **Pain points** | Frustraciones | No entiende el pricing, proceso lento, errores |
| **Oportunities** | Mejoras potenciales | Trial gratis, checkout simplificado, onboarding |

### Consejos:

- **Usa datos reales** — No specules, investiga
- **Sé específico** — Un journey por persona, no uno genérico
- **Captura emociones** — Usa la "escalera emocional" (frustrado → neutral → encantado)
- **Identifica los "momentos de la verdad"** — Puntos donde el usuario decide continuar o abandonar
- **Involucra al equipo** — El proceso de crear el mapa es tan valioso como el resultado

## 💡 Ejemplo de uso

### Contexto:
Una tienda online de decoración quiere entender por qué muchos usuarios abandonan el checkout.

### CJM creado:

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     CUSTOMER JOURNEY MAP                                │
│                   Usuario: María, 35 años, compra online frecuente     │
├─────────────────────────────────────────────────────────────────────────┤
│  ETAPA:    Conciencia      Consideración        Decisión        Uso    │
│                                                                          │
│  ACCIONES: Ve ads en IG   Visita el sitio    Compara          Recibe   │
│            keywords       brows categoría     opciones         producto│
│              \                \                  \               \      │
│               \    ___________V_______    _______V______    ___V___    │
│                \  /                  \  /             \  /        \    │
│  EMOCIONES:     :|                   :|              :|          :)   │
│              Curiosa             Emocionada        Decidida    Conten-  │
│                                                          /         ta   │
│  TOUCHPOINTS: Instagram      Web, email         Checkout       Email,  │
│               + Google          retargeting       (ABANDONO)   App     │
│                                                                          │
│  PAIN POINTS:                 No sabe si             Shipping      ---
│                            cabe en su            costoso       ---
│                            casa                  (~30%) abandona│
│                                                                          │
│  OPPORTUNITIES: Instagram     Medidas en           Calcular     ---
│ _shoppable_    producto          shipping         ---
│                   early         upfront            ---
│               |              |              |              |         |
└─────────────────────────────────────────────────────────────────────────┘
```

### Análisis derivado:
> **Punto de dolor crítico:** En la etapa de consideración, María no sabe las dimensiones exactas del producto y no puede visualizar si cabe en su espacio.
>
> **Oportunidad priorizada:** Agregar un "visualizador de tamaño" en la página de producto que muestre el producto a escala en una foto de un cuarto modelo.
>
> **Resultado:** Implementar esta feature redujo el abandono de checkout en un 18%.

## 📅 Cuándo usarlo

### Fases recomendadas:

- **Discover** — Para entender la experiencia actual del usuario con productos similares
- **Define** — Para identificar problemas específicos a resolver en el challenge
- **Develop** — Para diseñar soluciones que encajen naturalmente en el journey

### Situaciones ideales:

- ✅ Para entender por qué usuarios abandonan un proceso
- ✅ Para identificar gaps entre canales (omnicanal)
- ✅ Al diseñar nuevos productos o servicios
- ✅ Para mejorar la experiencia de usuario existente
- ✅ En workshops con stakeholders
- ✅ Para comunicar la experiencia del usuario al equipo

### No usar cuando:

- ❌ No tienes datos sobre el usuario (investiga primero)
- ❌ Solo necesitas entender un momento específico (usa otras herramientas)
- ❌ El producto es muy simple (overkill)
