# Mapa de Empatía (Empathy Map)

## ¿Qué es?

El Mapa de Empatía es una herramienta visual que ayuda a entender profundamente a los usuarios al capturar sus observaciones, emociones, pensamientos y acciones. Fue creado por Dave Gray.

## Cómo se usa

### Estructura del Mapa de Empatía:

```
┌──────────────────────────────────────────────────────────────────┐
│                          USUARIO                                │
├────────────────────┬────────────────────────────────────────────┤
│     VE             │            OYE                              │
│  Qué ve el        │  Qué escucha el                            │
│  usuario?         │  usuario?                                 │
│                    │                                            │
├────────────────────┼────────────────────────────────────────────┤
│     PIENSA         │            SIENTE                          │
│  Qué piensa el    │  Qué siente el                            │
│  usuario?         │  usuario?                                 │
│  (miedos,         │  (emoción, frustración,                    │
│   creencias)      │   motivación)                             │
├────────────────────┼────────────────────────────────────────────┤
│     DICE           │            HACE                             │
│  Qué dice el      │  Qué hace el                              │
│  usuario?         │  usuario?                                 │
│                    │                                            │
└────────────────────┴────────────────────────────────────────────┘
```

### Cuadrantes:

1. **VE (See)** — Qué ve en su entorno
2. **OYE (Hear)** — Qué escucha de otros
3. **PIENSA (Think)** — Qué pasa por su mente
4. **SIENTE (Feel)** — Qué emociones experimenta
5. **DICE (Say)** — Qué dice en voz alta
6. **HACE (Do)** — Cómo actúa

### Diferencia con Persona:

| Persona | Mapa de Empatía |
|---------|-----------------|
| Perfil estático | Momento específico |
| Datos demográficos | Insights行為ales |
| Una por producto | Múltiples por escenario |

## Cuándo usarlo

- En **Discover** después de entrevistas
- En **Define** para sintetizar insights
- En workshops de discovery
- Para validar assumptions
