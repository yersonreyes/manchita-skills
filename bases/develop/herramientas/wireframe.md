# Wireframe

## 📖 ¿Qué es?

El **Wireframe** es una representación esquemática de baja/media fidelidad que muestra la estructura y layout de una pantalla o flujo, sin detalles visuales. Es el "blueprint" del diseño — como los planos de una casa antes de construir.

La diferencia clave entre wireframe y mockup es la fidelidad: el wireframe se enfoca en la estructura, el layout, y la jerarquía de información. Los colores, tipografía, e imágenes vienen después. Esta separación permite validar la usabilidad sin distractores visuales.

El wireframe es el paso intermedio entre los sketchs en papel y el diseño de alta fidelidad. Permite iterate rápidamente porque es fácil de cambiar.

---

## 🔧 Cómo se usa

### Características del wireframe:

| Característica | Descripción |
|----------------|-------------|
| **Estructura primero** | Se enfoca en contenido y layout |
| **Sin estilo** | Colores, tipografía vienen después |
| **Rápido de crear** | Iteración fácil |
| **Foco en funcionalidad** | Qué hace cada elemento |

### Niveles de fidelidad:

| Nivel | Descripción | Tiempo | Cuándo usarlo |
|-------|-------------|--------|---------------|
| **Low-fi** | Boceto a mano o básico digital | 5-10 min | Para exploración inicial |
| **Mid-fi** | Digital con estructura clara | 30-60 min | Para validación de layout |
| **Hi-fi** | Casi parece mockup | 2-4 horas | Para handoff detallado |

### Elementos comunes en wireframes:

```
┌─────────────────────────────────────────────────────────┐
│ [LOGO]                              [BUSCAR] [CUENTA]  │  ← Header
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ╔═══════════════╗    ╔═══════════════╗               │
│  ║   BANNER     ║    ║   BANNER     ║               │  ← Hero Section
│  ╚═══════════════╝    ╚═══════════════╝               │
│                                                         │
│  [Card 1]  [Card 2]  [Card 3]  [Card 4]               │  ← Grid
│                                                         │
│  ─────────────── Contenido destacado ─────────────     │
│                                                         │
│  [Footer links]              [Social] [Newsletter]     │  ← Footer
└─────────────────────────────────────────────────────────┘
```

### Proceso:

1. **Definir objetivos** — Qué pantalla y para quién
2. **Sketch rápido** — A mano para explorar
3. **Digitalizar** — En la herramienta elegida
4. **Agregar contenido** — Texto e imágenes placeholders
5. **Review** — Con el equipo
6. **Iterar** — Basado en feedback

### Consejos:

- **No te distractes con el estilo** — Eso viene después
- **Usa representaciones estandarizadas** — iconos de librerías
- **Incluye notas** — Para decisiones de diseño
- **Itera rápido** — Los wireframes son descartables

## 💡 Ejemplo de uso

### Contexto:
Un equipo necesita diseñar la pantalla de inicio de una app de fitness.

### Wireframe creado:

```
WIREFRAME: App Fitness - Home
═══════════════════════════════════════════════════════════

      [LOGO]                  [NOTIFICATIONS] [PROFILE]
    ─────────────────────────────────────────────────
    
    ┌───────────────────────────────────────────┐
    │  Hola, Juan                                │
    │  💪 Tu meta: 5000 pasos hoy                │
    │  ▓▓▓▓▓▓▓░░░░░ 3200 / 5000 pasos           │
    └───────────────────────────────────────────┘

    ┌──────────────┐ ┌──────────────┐ ┌──────────┐
    │ 🏃 Running   │ │ 🧘 Yoga     │ │ 💪 Gym  │
    │ 3km hoy     │ │ 20 min      │ │ 45 min  │
    └──────────────┘ └──────────────┘ └──────────┘

    ──────── Ejercicios recomendados ──────────

    ┌─────────────────────────────────────┐
    │ [IMG]    Full Body Workout          │
    │          30 min • Intermedio        │
    │          [INICIAR]                   │
    └─────────────────────────────────────┘

    ┌─────────────────────────────────────┐
    │ [IMG]    Estiramientos              │
    │          15 min • Principiante       │
    │          [INICIAR]                   │
    └─────────────────────────────────────┘
```

### Elementos incluidos:
- Jerarquía clara (título, métricas, acciones)
- Representaciones de imágenes (rectángulos)
- Botones y CTAs claramente marcados
- Espaciado y alineación básicos

### Decisión:
> **Resultado:** El wireframe fue aprobado. Ahora puede convertirse en mockup de alta fidelidad.

## 📅 Cuándo usarlo

### Fases recomendadas:

- **Develop** — Para diseño de estructura

### Situaciones ideales:

- ✅ Para explorar múltiples layouts rápido
- ✅ Antes de invertir en diseño de alta fidelidad
- ✅ Para validar con usuarios sin sesgo visual
- ✅ Para handoff con desarrollo

### No usar cuando:

- ❌ Necesitas presentar a clientes externos (mockup mejor)
- ❌ El diseño es muy simple (no justifica)
