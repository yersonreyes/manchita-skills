# Prototipo Funcional

## ¿Qué es?

El Prototipo Funcional es una representación del producto que realmente "funciona", permitiendo a usuarios completar tareas reales. A diferencia de mockups estáticos, tiene comportamiento real y puede ejecutar flujos completos.

## Cómo se usa

### Propósito:

- Testear flujos reales con usuarios
- Validar que la solución funciona técnicamente
- Encontrar bugs y issues de UX
- Obtener feedback de experiencia de uso real

### Tipos de prototipos funcionales:

| Tipo | Descripción | Cuándo usarlo |
|------|-------------|---------------|
| MVP | Producto mínimo viable | Lanzamiento inicial |
| Pilot | Versión limitada | Testing en producción |
| Beta | Almost production | Validación amplia |
| Feature flag | Funcionalidad toggled | Testing gradual |

### Diferencia con otros prototipos:

| Tipo | Lo que permite |
|------|----------------|
| Mockup | Ver cómo se ve |
| Clickable | Navegar flujos |
| Funcional | Completar tareas reales |

### Herramientas para crear prototipos funcionales:

- **Figma + Prototype** — Interactivo
- **Principle / After Effects** — Animaciones
- **Protopie** — Interacciones complejas
- **Codepen / Codesandbox** — Prototipos web
- **Flutter / React Native** — Apps funcionales

### Proceso:

1. **Definir flujos críticos** — Qué debe funcionar
2. **Priorizar features** — Solo lo esencial
3. **Desarrollar** — Build el prototipo
4. **Testear** — Con usuarios reales
5. **Iterar** — Bug fixes y mejoras

## Cuándo usarlo

- En **Develop** cuando se necesita validación real
- Para pasar de diseño a desarrollo
- En testing de usabilidad avanzado
- Para validación de negocio

## 📖 ¿Qué es?

El **Prototipo Funcional** es una representación del producto que realmente "funciona", permitiendo a usuarios completar tareas reales con comportamiento automático. Es un paso más allá de los mockups interactivos porque tiene lógica real, no solo navegación.

A diferencia de los prototipos de baja fidelidad, el funcional requiere desarrollo real (aunque mínimo). Permite testear no solo la usabilidad sino también la viabilidad técnica del producto.

## 💡 Ejemplo de uso

**Caso: Testear nueva funcionalidad de búsqueda en ecommerce**

**Desarrollo del prototipo:**
- Backend: API básica con dummy data
- Frontend: Interface funcional con búsqueda real
- Base de datos: 50 productos con metadata

**Testing con 10 usuarios:**
1. Task 1: "Encontrar camisa azul debajo de $50"
2. Task 2: "Encontrar regalo para alguien que le gusta correr"
3. Task 3: "Comparar 3 productos similares"

**Hallazgos funcionales:**
- El filtro de precio no funcionaba correctamente
- La búsqueda no entendía sinónimos ("running" = "correr")
- Los resultados no se podían ordenar por rating

**Hallazgos de UX:**
- Los usuarios expected filtering por color más visible
- Querían ver reseñas en los resultados
- La opción de "guardar búsqueda" era necesaria

**Resultado:** 15 bugs/mejoras identificados antes de desarrollo completo. El equipo ahorró semanas de iteración.
