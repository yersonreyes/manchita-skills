# From-To (De-A)

## 📖 ¿Qué es?

El diagrama **From-To** (también conocido como De-A o Current-State-to-Future-State) es una herramienta de síntesis que muestra la transformación o evolución de un estado actual (From) a un estado futuro (To). Es una de las herramientas más poderosas para visualizar el cambio y las brechas que necesitan cerrarse.

Esta herramienta es fundamental en la fase de Define porque ayuda a cristalizar la visión del problema y la solución. A diferencia de otras herramientas que se enfocan en entender el presente, el From-To se enfoca en el **futuro** — te obliga a pensar a dónde quieres llegar y qué necesitas hacer para llegar ahí.

El origen de esta herramienta está en el pensamiento de diseño estratégico y la planificación de productos, donde se necesita comunicar claramente la transformación que un producto o servicio va a crear.

---

## 🔧 Cómo se usa

### Paso a paso:

1. **Define el problema** — ¿Cuál es el estado actual (From)?
2. **Define la visión** — ¿Cuál es el estado futuro (To)?
3. **Mapea las brechas** — ¿Qué falta para hacer la transición?
4. **Identifica los pasos** — ¿Cómo se hace la transición?
5. **Establece el timeline** — ¿En qué orden?

### Estructura del diagrama:

```
┌─────────────────────────────────────────────────────────────────┐
│                         FROM → TO                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│    FROM (Estado Actual)           TO (Estado Futuro)            │
│    ─────────────────────          ──────────────────           │
│                                                                  │
│    • Problema 1              →     • Solución 1                   │
│    • Problema 2              →     • Beneficio 2                  │
│    • Pain point X            →     • Oportunidad Y               │
│                                                                  │
│    [Contexto actual]                 [Visión de futuro]         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### Consejos:

- **Sé específico** — "El usuario tiene que ir al banco" > "El proceso es lento"
- **Piensa en transformación** — No solo解决问题, piensa en el cambio de vida
- **Usa datos reales** — Basado en investigación, no suposiciones
- **Incluye contexto** — El From y To deben ser entendibles por cualquiera

## 💡 Ejemplo de uso

### Contexto:
Una fintech quiere definir la visión para una nueva app de pagos para freelancers.

### From-To creado:

```
┌─────────────────────────────────────────────────────────────────┐
│              FROM → TO: App de Pagos para Freelancers            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│    FROM (Estado Actual)           TO (Estado Futuro)             │
│    ─────────────────────          ──────────────────           │
│                                                                  │
│    • Cobrar tarda 3-7 días      →     • Cobrar en segundos        │
│    • Ir al banco a depositar   →     • Todo desde el celular     │
│    • Comisiones 3-5% por       →     • Comisiones 0.5%           │
│      transacción                                           │
│    • No hay manera de          →     • Facturas automáticas      │
│      facturar                                               │
│    • No hay histórico de       →     • Historial completo        │
│      ingresos                                             │
│                                                                  │
│    Contexto: El freelancer actualmente pierde hasta 3 días      │
│    esperando pagos y 5% en comisiones. No tiene manera         │
│    de generar invoices profesionales.                          │
│                                        │
│    Visión: Un freelancer puede cobrar, facturar y gestionar    │
│    sus ingresos 100% desde el celular, con comisiones          │
│    mínimas y sin ir al banco.                                  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### Uso derivado:
> **Para roadmap:** Las features del To se priorizan en el backlog.
> **Para comunicación:** El From-To se comparte con stakeholders para alinear visión.
> **Para validación:** Cada feature debe contribuir al cierre de brechas From-To.

## 📅 Cuándo usarlo

### Fases recomendadas:

- **Define** — Para definir la visión del producto/servicio
- **Develop** — Para validar que las soluciones van hacia el To

### Situaciones ideales:

- ✅ Para comunicación con stakeholders
- ✅ Para roadmap planning
- ✅ En workshops de estrategia
- ✅ Al definir un nuevo producto o feature
- ✅ Para comunicar la propuesta de valor
- ✅ Para validar que vas en la dirección correcta

### No usar cuando:

- ❌ No tienes claro el problema (investiga primero)
- ❌ El cambio es muy complexo (divide en múltiples From-To)
- ❌ Solo necesitas entender el presente (usa otras herramientas)
