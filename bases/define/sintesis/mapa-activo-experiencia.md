# Mapa Activo de la Experiencia (Active Experience Map)

## 📖 ¿Qué es?

El **Mapa Activo de la Experiencia** es una versión dinámica del Customer Journey Map que no solo muestra las etapas, sino también los momentos de interacción activa, los touchpoints específicos, y las oportunidades de intervención. A diferencia del Journey Map tradicional (que es más pasivo), el Mapa Activo te dice exactamente dónde y cómo puedes actuar.

La diferencia clave está en que el Mapa Activo va más allá de entender el journey — te dice específicamente qué hacer en cada touchpoint. Es más prescriptive, no solo descriptive.

Esta herramienta es particularmente útil cuando ya tienes un Journey Map pero necesitas pasar a la acción — identificar los momentos específicos donde puedes intervenir para mejorar la experiencia.

---

## 🔧 Cómo se usa

### Estructura:

```
┌────────────────────────────────────────────────────────────────────────┐
│                MAPA ACTIVO DE LA EXPERIENCIA                           │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│  ETAPA:     CONOCIMIENTO    DECISIÓN    USO       POST-USO             │
│             ────────────   ─────────  ──────    ──────────             │
│                                                                        │
│  ACCIONES:  Busca info     Compara    Usa el    Comparte              │
│             del problema   opciones   producto  experiencia            │
│                                     ↓                                  │
│  TOUCHPOINTS:    Web        Reviews    App      Redes                 │
│                  │            │          │         │                  │
│                  ▼            ▼          ▼         ▼                  │
│  MOMENTOS        [Descubrim]  [Evaluac] [Entrega] [Advocacy]          │
│  CLAVE:                                                                 │
│                                                                        │
│  OPORTUNIDADES: SEO        Trial      Onboard   Referal               │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

### Diferencia con Journey Map tradicional:

| Aspecto | Journey Map | Mapa Activo |
|---------|-------------|-------------|
| Enfoque | Pasivo | Acciones activas |
| Estructura | Lineal | Con branching |
| Profundidad | Solo etapas | Con touchpoints específicos |
| Output | Pain points | + oportunidades de intervención |
| Uso | Entender | Actuar |

### Paso a paso:

1. **Parte del Journey Map** — Si ya tienes uno, úsalo como base
2. **Identifica touchpoints** — Dónde toca el usuario el producto
3. **Define momentos clave** — Los "momentos de la verdad"
4. **Documenta oportunidades** — En cada touchpoint, qué puedes hacer
5. **Prioriza** — Qué oportunidades son más impactantes

### Consejos:

- **No confundas con journey map** — Es más accionable
- **Sé específico** — Los touchpoints deben ser concretos
- ** Incluye momentos de no-uso** — Qué pasa entre interacciones
- **Mapea los hánd-offs** — Entre canales, entre equipos

## 💡 Ejemplo de uso

### Contexto:
Un equipo tiene un Journey Map del usuario de su app de delivery y quiere convertirlo en un Mapa Activo para priorizar features.

### Journey Map existente:
```
Conocimiento → Consideración → Decisión → Uso → Post-uso
```

### Conversión a Mapa Activo:

```
MAPA ACTIVO: APP DE DELIVERY
═══════════════════════════════════════════════════════════════════════

ETAPA:         CONOCIMIENTO      DECISIÓN        USO          POST-USO
               ─────────────    ────────────   ────────     ─────────

ACCIONES:      Busca opciones    Compara         Hace        Comparte
               en apps          restaurantes     pedido      experiencia

TOUCHPOINTS:   App Store         Reviews         App         Redes
               ├─ Banner          ├─ Ratings       ├─ Menu     ├─ Instagram
               ├─ Search          ├─ Fotos         ├─ Carrito  ├─ WhatsApp
               └─ Social          └─ Comentarios  ├─ Pago     └─ Reseña

MOMENTOS       "Descubrimiento"  "Evaluación"    "Entrega"   "Advocacy"
CLAVE:         (primera vez)     (comparar)     (crucial)   (compartir)

OPORTUNIDADES:
├─ SEO: Estar donde el usuario busca "comida cerca"
├─ Reviews: Incentivar fotos de platos
├─ Onboarding: Reducir fricción en primer pedido
├─ Tracking: Notificaciones proactivas de status
└─ Referral: Sistema de referidos para post-uso
```

### Análisis para roadmap:

| Touchpoint | Oportunidad | Prioridad | Impacto |
|------------|-------------|-----------|---------|
| Onboarding | Reducir pasos de registro | Alta | Alto |
| Menú | Fotos más grandes y真实的 | Alta | Medio |
| Tracking | Notificaciones proactivas | Alta | Alto |
| Post-entrega | Incentivar reseñas con puntos | Media | Medio |

### Output:
> **Insight clave:** El momento de "decisión" es el más competitivo — el usuario está comparando entre varias apps. La oportunidad está en diferenciarse con fotos de platos más realistas y reviews con información útil.

## 📅 Cuándo usarlo

### Fases recomendadas:

- **Define** — Para identificar momentos clave
- **Develop** — Para diseñar intervenciones específicas
- **Deliver** — Para roadmap de producto

### Situaciones ideales:

- ✅ Para diseñar intervenciones específicas
- ✅ En análisis de experiencia de usuario
- ✅ Para roadmap de producto
- ✅ Cuando ya tienes un Journey Map

### No usar cuando:

- ❌ No tienes un Journey Map base (haz ese primero)
- ❌ Solo necesitas entender la experiencia ( Journey Map es suficiente)
