# Mapa de Empatía (Empathy Map)

## 📖 ¿Qué es?

El **Mapa de Empatía** es una herramienta visual creada por **Dave Gray** que ayuda a entender profundamente a los usuarios al capturar lo que ven, escuchan, piensan, sienten, dicen y hacen. A diferencia de la Persona (que es un perfil estático), el Mapa de Empatía captura un momento específico y las dimensiones emocionales y comportamentales del usuario.

Esta herramienta es fundamental en Design Thinking porque establece la empatía como base del proceso. Sin ella, corremos el riesgo de diseñar soluciones que no resonan con las necesidades reales del usuario.

El Mapa de Empatía se utiliza típicamente después de hacer investigación con usuarios (entrevistas, observaciones) y antes de crear las Personas. Es un paso de síntesis que transforma datos crudos en insights accionables.

---

## 🔧 Cómo se usa

### Paso a paso:

1. **Selecciona un escenario específico** — No para "todos los usuarios", sino para un momento/context específico
2. **Usa datos reales** — Entrevistas, observaciones, analytics
3. **Llena cada cuadrante** — Ve, Oye, Piensa, Siente, Dice, Hace
4. **Busca patrones** — ¿Qué es consistente? ¿Qué contradice?
5. **Deriva insights** — Conclusiones que guían el diseño

### Los 6 cuadrantes:

| Cuadrante | Preguntas guía | Ejemplo |
|-----------|---------------|---------|
| **VE** | ¿Qué ve en su entorno? ¿Qué productos usa? ¿A quién tiene alrededor? | "Ve que sus competidores ofrecen planes más baratos" |
| **OYE** | ¿Qué escucha de amigos, familia, medios? ¿Qué le dicen? | "Escucha que todos sus amigos dicen que el pricing es muy caro" |
| **PIENSA** | ¿Qué pasa por su mente? ¿Qué le preocupa? ¿Qué cree? | "Piensa que no vale la pena pagar más porFeatures que no usa" |
| **SIENTE** | ¿Qué emociones experimenta? ¿Frustración, ansiedad, esperanza? | "Siente frustración cuando el proceso de checkout toma mucho tiempo" |
| **DICE** | ¿Qué dice en voz alta? ¿Qué palabras usa? | "Dice que 'solo quiero algo que funcione y ya'" |
| **HACE** | ¿Cómo actúa? ¿Qué comportamiento observable tiene? | "Hace scroll rápidamente por las opciones, no lee detalles" |

### Diferencia con Persona:

| Característica | Persona | Mapa de Empatía |
|----------------|---------|-----------------|
| Temporalidad | Perfil general | Momento específico |
| Tipo de datos | Demográfico, comportamiento | Emocional, observacional |
| Cantidad | Una por producto | Múltiples por escenario |
| Uso | Guía de diseño | Síntesis post-investigación |

### Consejos:

- **Sé observacional** — Lo que hace el usuario > lo que dice
- **Diferencia lo que dice vs hace** — A menudo son diferentes
- **No juzgues** — Solo documenta, no interpretes aún
- **Usa verbos** — Describe acciones, no características

## 💡 Ejemplo de uso

### Contexto:
Después de hacer 10 entrevistas con freelancers que usan una app de productividad, el equipo quiere sintetizar los insights.

### Mapa de Empatía creado:

```
┌─────────────────────────────────────────────────────────────────────┐
│            MAPA DE EMPATÍA - Freelancer usando app de productivity │
│                     Contexto: Fin de mes, gestionando proyectos    │
├────────────────────┬───────────────────────────────────────────────┤
│     VE             │            OYE                                   │
│ • Ve notificaciones│ • Escucha de compañeros que                    │
│   de múltiples      │   usan Notion y Asana                         │
│   apps (Slack,     │ • Escuchó en un podcast                       │
│   Email, Trello)   │   que "la multitarea es mala"                  │
│ • Ve su lista de   │ • Su manager dice "prioriza lo                │
│   tareas creciente  │   urgente, no lo importante"                  │
├────────────────────┼───────────────────────────────────────────────┤
│     PIENSA         │            SIENTE                              │
│ • "Si no lo anoto, │ • Siente AGOTAMIENTO al ver                    │
│   se me va a olvidar"│   todas las tareas pendientes                │
│ • "¿Por qué no     │ • Siente CULPA cuando no                      │
│   puedo hacer esto │   termina algo a tiempo                        │
│   más rápido?"     │ • Siente FRUSTRACIÓN                          │
│ • "Estoy perdiendo  │   cuando cambia de app para                   │
│   tiempo en esto"  │   cada cosa                                   │
├────────────────────┼───────────────────────────────────────────────┤
│     DICE           │            HACE                                │
│ • "No me da la     │ • Abre la app 15+ veces al día                 │
│   vida, me da más  │ • Tiene 50+ tabs abiertos                      │
│   trabajo"         │ • Context switching cada 5 min                │
│ • "Necesito algo   │ • Pone timestamps en sus                       │
│   que simplifique" │   notas para "no olvidar"                      │
│ • "Esto es un      │ • Trabaja horas extra para                     │
│   desastre"        │   compensar "pérdida de tiempo"               │
└────────────────────┴───────────────────────────────────────────────┘
```

### Insights derivados:
> **Insight 1:** El freelancer no necesita más features; necesita **menos contexto switching**.
>
> **Insight 2:** La culpa y el agotamiento son emociones dominantes. El diseño debe ser **calmante**, no estimulante.
>
> **Insight 3:** "No me da la vida" sugiere que la app se percibe como una carga adicional, no como ayuda.

## 📅 Cuándo usarlo

### Fases recomendadas:

- **Discover** — Para sintetizar insights después de entrevistas/observaciones
- **Define** — Para informar el Point of View y How Might We

### Situaciones ideales:

- ✅ Después de hacer investigación cualitativa (entrevistas, ethnografía)
- ✅ Para validar o crear una Persona
- ✅ En workshops de discovery con el equipo
- ✅ Cuando tienes datos contradictorios entre lo que el usuario dice y hace
- ✅ Para comunicar insights a stakeholders

### No usar cuando:

- ❌ No tienes datos de usuarios (no es para specular)
- ❌ Solo necesitas un perfil demográfico (usa Persona)
- ❌ El contexto es muy amplio (divide por momentos específicos)
