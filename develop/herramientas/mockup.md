# Mockup

## 📖 ¿Qué es?

El **Mockup** es una representación visual de alta fidelidad del producto que muestra cómo se verá el diseño final. A diferencia del wireframe (que es más básico y estructural), el mockup incluye estilos, colores, tipografía, imágenes, y todos los detalles visuales.

El mockup es lo más cercano al producto final sin ser interactivo. Es la herramienta que se usa para presentar el diseño a stakeholders, clientes, y al equipo de desarrollo como spec de lo que necesitan construir.

Los mockups son especialmente importantes porque permiten visualizar el producto antes de invertir en desarrollo, y aseguran que todos los stakeholders tengan la misma expectativa visual.

---

## 🔧 Cómo se usa

### Características del mockup:

| Característica | Descripción |
|----------------|-------------|
| **Alta fidelidad** | Se parece mucho al producto final |
| **Estático** | No es interactivo (a menos que sea clickable) |
| **Detallado** | Incluye todos los elementos visuales |
| **Presentable** | Listo para enseñar a clientes y stakeholders |

### Niveles de fidelidad:

| Nivel | Descripción | Cuándo usarlo |
|-------|-------------|---------------|
| **Baja (wireframe)** | Blanco y negro, sin estilo | Para estructura y layout |
| **Media** | Con color, sin imágenes reales | Para validar dirección visual |
| **Alta (mockup)** | Completo, con imágenes | Para presentación a stakeholders |
| **Interactivo** | Clickable, con transiciones | Para testing de usuario |

### Herramientas para crear mockups:

| Herramienta | Plataforma | Precio |
|-------------|------------|--------|
| **Figma** | Web, Windows, Mac | Gratis |
| **Sketch** | Mac | Prueba gratis |
| **Adobe XD** | Windows, Mac | Prueba gratis |
| **Adobe Photoshop** | Windows, Mac | De pago |
| **Canva** | Web | Gratis |
| **Balsamiq** | Web, Windows, Mac | Prueba gratis |

### Proceso:

1. **partir del wireframe** — Ya tienes la estructura
2. **Aplicar design system** — Colores, tipografía, componentes
3. **Agregar contenido** — Imágenes, texto real
4. **Revisar consistencia** — Asegurar coherencia visual
5. **Exportar para review** — En formatos apropiados

### Consejos:

- **Usa design system** — Mantén consistencia
- **Contenido real es mejor** — No Lorem Ipsum
- **Incluye estados** — Hover, active, disabled
- **Anota decisiones** — Para comunicar al equipo

## 📅 Cuándo usarlo

### Fases recomendadas:

- **Develop** — Para diseño detallado
- **Deliver** — Para handoff a desarrollo

### Situaciones ideales:

- ✅ Para presentar a stakeholders y clientes
- ✅ Para handoff al equipo de desarrollo
- ✅ Para testing con usuarios (con interactividad)
- ✅ Para validar dirección visual

### No usar cuando:

- ❌ Necesitas validar estructura (usa wireframe)
- ❌ El tiempo es muy limitado (prototipo rápido primero)
