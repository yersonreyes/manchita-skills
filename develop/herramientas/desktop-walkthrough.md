# Desktop Walkthrough

## ¿Qué es?

El Desktop Walkthrough es una técnica de revisión donde el equipo revisa un diseño caminando paso a paso a través del flujo, como si fuera el usuario. Se hace "en la mesa" (desktop), sin usuarios reales.

## Cómo se usa

### Proceso:

1. **Presentar el diseño** — Wireframe, mockup o prototipo
2. **Recorrer el flujo** — Paso a paso como usuario
3. **Identificar issues** — Confusiones, gaps, errores
4. **Documentar** — Mejoras a hacer

### Usos:

- Revisión de diseño interna
- Encontrar issues antes de testing
- Alineación del equipo
