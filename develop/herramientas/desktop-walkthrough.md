# Desktop Walkthrough

## ¿Qué es?

El Desktop Walkthrough es una técnica de revisión donde el equipo revisa un diseño caminando paso a paso a través del flujo, como si fuera el usuario. Se hace "en la mesa" (desktop), sin usuarios reales.

## Cómo se usa

### Proceso:

1. **Presentar el diseño** — Wireframe, mockup o prototipo
2. **Recorrer el flujo** — Paso a paso como usuario
3. **Identificar issues** — Confusiones, gaps, errores
4. **Documentar** — Mejoras a hacer

### Usos:

- Revisión de diseño interna
- Encontrar issues antes de testing
- Alineación del equipo

## 📖 ¿Qué es?

El Desktop Walkthrough es una técnica de validación interna donde el equipo de proyecto "camina" a través del diseño como si fueran el usuario, sin necesidad de usuarios reales. Es como hacer un "ensayo" del recorrido del usuario sentado frente a la pantalla (desktop), simulando cada paso y tomando notas de cualquier confusión o problema que encuentren. Es una revisión de diseño rápida y económica antes de invertir en testing con usuarios reales.

## 💡 Ejemplo de uso

**Caso:** Tienes un nuevo flujo de checkout para tu e-commerce. Para hacer un desktop walkthrough:
1. Reúnete con el equipo de diseño y producto (4-6 personas)
2. Proyecta el diseño en pantalla o abre el prototipo
3. Una persona "narra" el recorrido: "Yo soy el usuario, voy a buscar un producto, lo agrego al carrito..."
4. En cada paso, todos señalan con la mano o dicen "STOP" si ven un problema
5. Documentan cada issue en un post-it o documento
6. Al final, priorizan los problemas encontrados

En 30-45 min puedes encontrar 15-20 issues que luego corriges antes de testing más formales.
