# Diseño de Escenarios

## ¿Qué es?

El Diseño de Escenarios es una técnica que crea narrativas detalladas sobre cómo un usuario interactúa con un producto o servicio en situaciones específicas. Ayuda a visualizar y validar soluciones en contextos realistas.

## Cómo se usa

### Estructura de un escenario:

```
DISEÑO DE ESCENARIOS
├── Contexto
│   ├── Quién es el usuario
│   ├── Dónde está
│   ├── Cuándo usa el producto
│   └── Qué está tratando de lograr
├── Flujo
│   ├── Paso 1: Acción
│   ├── Paso 2: Acción
│   └── Resultado
├── Emotions
│   ├── Qué siente en cada paso
│   └── Dónde se frustra/satisface
└── Oportunidades
    ├── Dónde podemos ayudar
    └── Dónde fallamos
```

### Tipos de escenarios:

| Tipo | Cuándo usarlo |
|------|---------------|
| Happy path | Flujo ideal, qué sale bien |
| Edge case | Situaciones límite |
| Error | Qué pasa cuando falla |
| Contextual | Situaciones específicas (viaje, emergencia) |

### Ejemplo:

```
ESCENARIO: Juan pide comida desde el trabajo
────────────────────────────────────────────────
CONTEXTO: 7pm, Juan en oficina, hambre, no quiere salir
FLUJO:
1. Abre app → ve recomendaciones
2. Filtra por "vegano" → ve opciones
3. Elige restaurante → revisa menu
4. Pide → paga → recibe confirmación
5. 35 min después → llega el delivery

EMOCIONES:
- Descubrir → curiosidad
- Elegir → satisfacción
- Esperar → ansiedad
- Recibir → alegría

OPORTUNIDADES:
- Recomendaciones personalizadas
- Tracking en tiempo real
- Notificaciones proactivas
```

## Cuándo usarlo

- En **Develop** para validar ideas
- Para diseñar flujos de usuario
- Para identificar pain points
- En prototipado
