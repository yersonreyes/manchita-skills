# Mapa Activo de la Experiencia (Develop)

## ¿Qué es?

El Mapa Activo de la Experiencia en la fase de Desarrollo es una herramienta de diseño que mapea todos los touchpoints y momentos de interacción del usuario con el producto, identificando oportunidades de diseño y optimización.

## Cómo se usa

### Estructura:

```
┌────────────────────────────────────────────────────────────────────────┐
│                MAPA ACTIVO DE LA EXPERIENCIA                           │
├────────────────────────────────────────────────────────────────────────┤
│                                                                        │
│  ETAPA:     CONOCIMIENTO    DECISIÓN    USO       POST-USO             │
│                                                                        │
│  ACCIONES:  Busca info     Compara    Usa el    Comparte              │
│             del problema   opciones   producto  experiencia            │
│                                     ↓                                  │
│  TOUCHPOINTS:    Web        Reviews    App      Redes                  │
│                  │            │          │         │                  │
│                  ▼            ▼          ▼         ▼                  │
│  MOMENTOS        [Descubrim]  [Evalac]  [Entrega] [Advocacy]           │
│  CLAVE:                                                                 │
│                                                                        │
│  OPORTUNIDADES: SEO        Trial      Onboard   Referal               │
│                                                                        │
│  SOLUCIONES:                                                                 │
│  - [Idea 1 para optimizar este momento]                                │
│  - [Idea 2 para este otro momento]                                     │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

### Proceso:

1. **Mapear etapas** — Del journey completo
2. **Identificar touchpoints** — Puntos de contacto
3. **Detectar oportunidades** — Dónde podemos mejorar
4. **Diseñar soluciones** — Ideas para cada momento
5. **Conectar con ideas** — De brainstorming previo

## Cuándo usarlo

- En **Develop** para diseñar experiencia
- Para validar que las ideas cubren el journey
- En prototipado de flujos
- Para encontrar gaps en la solución
