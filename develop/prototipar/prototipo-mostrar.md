# Prototipo para Mostrar

## ¿Qué es?

El Prototipo para Mostrar es una representación del producto diseñada específicamente para comunicar, presentar y obtener validación de stakeholders, clientes o usuarios. Su objetivo es contar una historia convincente y obtener feedback directed.

## Cómo se usa

### Propósito:

- Presentar a stakeholders/klientes
- Validar con usuarios potenciales
- Obtener buy-in y aprobación
- Comunicar la visión del producto

### Niveles de presentación:

| Nivel | Descripción | Audiencia |
|-------|-------------|-----------|
| Demo estático | Screenshots, mockups | Stakeholders |
| Demo interactivo | Clickable prototype | Usuarios, clientes |
| Demo con video | Video del flujo | Todos |
| MVP funcional | Producto funcional | Beta users |

### Características de un buen prototipo para mostrar:

1. **Narra una historia** — Cuenta el journey completo
2. **Highlights los beneficios** — Enfoca en valor
3. **Es interactivo** — Permite "jugar" con él
4. **Es pulido** — Se ve professional
5. **Anticipa preguntas** — Tiene respuestas listas

### Proceso:

1. **Definir audiencia** — A quién vas a mostrar
2. **Elegir el nivel** — Cuán detallado
3. **Craft el mensaje** — Qué quieres comunicar
4. **Crear el prototipo** — Pulido y presentable
5. **Preparar el pitch** — Cómo lo vas a introducir

## Cuándo usarlo

- En **Develop** para presentar a stakeholders
- Para obtener approval de presupuesto
- En ventas o pitches de producto
- Para validación con usuarios

## 📖 ¿Qué es?

El **Prototipo para Mostrar** es una representación del producto diseñada específicamente para comunicar, presentar y obtener validación de stakeholders, clientes o usuarios. A diferencia de los prototipos de exploración, este debe ser polished y narrativo.

Su objetivo es contar una historia convincente sobre el producto, no solo mostrar features. Debe responder a la pregunta "¿por qué esto importa?" de forma emocional y racional. Es una herramienta de venta interna y externa.

## 💡 Ejemplo de uso

**Caso: Pitch para conseguir presupuesto de desarrollo**

**Situación:** Tienes una idea de app pero necesitas aprobación del board para presupuesto.

**Prototipo para mostrar:**
1. **Video de 60 segundos** mostrando el journey completo del usuario
2. **Mockups de alta fidelidad** en Figma con interactividad
3. **Landing page** con价值的 proposición

**Contenido del pitch:**
- "Problema" (2 slides): Datos del dolor del usuario
- "Solución" (1 slide): Demo del prototipo
- "Impacto" (1 slide): Métricas esperadas
- "Solicitud" (1 slide): Cuánto y para qué

**Resultado:** El board pudo "ver" el producto antes de comprometerse. Tuvieron preguntas concretas en lugar de objeciones vagas. La aprobación fue más rápida porque podían visualizar el resultado.
