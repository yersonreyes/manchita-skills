# Prototipo Rápido

## ¿Qué es?

El Prototipo Rápido es una herramienta de diseño que crea representaciones veloces de una idea para obtener feedback inmediato. El objetivo es aprender rápido, fallar barato y iterar antes de invertir más recursos.

## Cómo se usa

### Principios:

- **Tiempo limitado** — Hours, no días
- **Materiales simples** — Papel, post-its, lo que haya
- **Iteración rápida** — Versiones sucesivas
- **Feedback inmediato** — Testear lo antes posible

### Técnicas de prototipado rápido:

| Técnica | Tiempo | Material |
|---------|--------|----------|
| Sketch en servilleta | 2 min | Papel |
| Paper prototype | 15 min | Papel, tijeras |
| Wizard of Oz | 30 min | Humans simulando |
| Clickable mockup | 1-2 hrs | Figma |
| MVP code | 1-2 días | Código |

### Proceso:

```
1. DEFINIR PREGUNTA   → Qué quiero validar?
2. CREAR RÁPIDO      → Mínimo necesario
3. TESTEAR           → Con usuarios reales
4. APRENDER          → Qué funcionó/no funcionó
5. ITERAR O PIVOT    → Continuar o cambiar
```

### Reglas del prototipado rápido:

- **Happy path primero** — Solo el flujo principal
- **Lo justo y necesario** — No sobre-diseñar
- **Baja fidelidad acceptable** — Funciona para learning
- **Destruible** — No te aferres al prototipo

## Cuándo usarlo

- En **Develop** para validation velocity
- Para explorar muchas ideas rápido
- En design sprints
- Cuando el tiempo es limitado
