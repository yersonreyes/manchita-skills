# Prototipo para Empatizar

## ¿Qué es?

El Prototipo para Empatizar es una herramienta de diseño que crea representaciones tangibles o digitales del producto para entender mejor las necesidades, emociones y contexto del usuario. No busca validar funcionalidad, sino generar empatía.

## Cómo se usa

### Propósito:

- Entender el problema desde la perspectiva del usuario
- Sentir lo que el usuario siente
- Identificar моменты de fricción emocionales
- Validar assumptions antes de invertir en desarrollo

### Tipos:

| Tipo | Descripción | Fidelidad |
|------|-------------|-----------|
| Role-play | Actuar la experiencia | Baja |
| Bodystorming | Prototipar con el cuerpo | Baja |
| Environmental | Recrear el entorno | Media |
| Experiencial | Vivir la experiencia | Media-Alta |

### Ejemplos:

- **Role-play:** Simular ser un cliente en una tienda
- **Bodystorming:** Hacer cola para sentir la frustración
- **Environmental:** Replicar el escritorio del usuario

### Proceso:

1. **Definir qué empathy** — Qué queremos sentir
2. **Elegir el método** — Rol-play, environment, etc.
3. **Crear el prototipo** — Rápido y simple
4. **Interactuar** — Vivir la experiencia
5. **Documentar insights** — Lo que aprendimos

## Cuándo usarlo

- En **Develop** antes de diseñar soluciones
- Para validar que entendemos al usuario
- Cuando hay disconnect con el usuario
- En investigación de nuevos dominios

## 📖 ¿Qué es?

El **Prototipo para Empatizar** es una herramienta de design thinking que crea representaciones tangibles o digitales del producto con el único propósito de entender mejor las necesidades, emociones y contexto del usuario. A diferencia de otros prototipos, no busca validar funcionalidad sino validar comprensión.

El objetivo es que el equipo de diseño "sienta" lo que el usuario siente, identifique momentos de fricción emocional, y valide assumptions antes de invertir recursos en desarrollo. Es empático por naturaleza, no evaluativo.

## 💡 Ejemplo de uso

**Caso: Diseñar experiencia de checkout para adultos mayores**

**Problema:** El equipoyoung no entiende por qué los usuarios mayores abandonan el checkout.

**Solución - Prototipo experiencial:**
1. Simular limitaciones visuales: usar glasses que reducen visión periférica
2. Simular limitaciones motoras: guantes que dificultan movimientos finos
3. Ponerse en una situación de estrés: ruido, presión de tiempo
4. Intentar completar un checkout típico

**Resultado:** El equipo entendió que:
- Los botones pequeños son frustrantes
- Los campos de contraseña con symbols confunden
- El timeout del sesión genera ansiedad
- La verificación de tarjeta con muchos pasos abruma

Estas insights no habrían surgido en entrevistas tradicionales.
