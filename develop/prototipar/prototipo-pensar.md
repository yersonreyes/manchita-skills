# Prototipo para Pensar

## ¿Qué es?

El Prototipo para Pensar es una herramienta de diseño que usa representaciones tangibles del producto para explorar, iterar y validar ideas antes de comprometer recursos en desarrollo. Es un "think with your hands" - pensar con las manos.

## Cómo se usa

### Propósito:

- Explorar múltiples soluciones rápidamente
- Iterar sobre ideas sin código
- Comunicar ideas al equipo
- Identificar problemas antes de que sean costosos

### Tipos:

| Tipo | Descripción | Cuando usarlo |
|------|-------------|---------------|
| Sketch | Dibujo rápido en papel | Idea inicial |
| Wireframe | Estructura básica, sin estilo | Estructura de información |
| Storyboard | Secuencia de pantallas | Flujos de usuario |
| Paper prototype | Interactivo en papel | Testing temprano |

### Ejemplo de evolución:

```
Sketch → Wireframe → Alta fidelidad → Código
(5 min)  (30 min)     (2 horas)       (días)
```

### Proceso iterativo:

```
1. Idear → 2. Prototipar → 3. Testear → 4. Iterar
    ↑                                            │
    └──────────────── Feedback ─────────────────┘
```

### Herramientas:

- **Baja fidelidad:** Papel, post-its, marker
- **Media:** Figma, Balsamiq, Sketch
- **Alta:** Figma, Adobe XD, Principle

## Cuándo usarlo

- En **Develop** para explorar ideas
- Para comunicar conceptos al equipo
- Antes de development costoso
- En design sprints
