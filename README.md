## Acerca de

Este repo documenta herramientas utilizadas en las diferentes fases del proceso de diseño, siguiendo la metodología **Double Diamond**:

```
Discover → Define → Develop → Deliver
```

Las herramientas aquí documentadas no son herramientas digitales de diseño (Figma, Sketch, etc.), sino recursos, métodos y herramientas para el proceso de diseño en general.

> 📖 Basado en **Designpedia** — Libro de referencia para herramientas y métodos de diseño

## 🔍 Discover (Descubrir)

**Objetivo:** Investigar, empatizar y entender el problema

En esta fase inicial, el objetivo es comprender a fondo el problema y el contexto antes de buscar soluciones. Se investiga a los usuarios, el mercado y la competencia.

### Subcategorías:

| Icono | Subcategoría | Descripción |
|-------|--------------|-------------|
| 🏢 | **Empresa** | Análisis interno del negocio |
| 👥 | **Cliente/Mercado** | Investigación del usuario y mercado |

### Herramientas disponibles:

- 5 Porqués, Business Model Canvas, FODA, Diagnóstico de Industria, Análogos/Antilogos, Diagrama de Sistema, In/Out
- Stakeholder Map, Role Play, Persona, Mapa de Empatía, Customer Journey Map

---

## 🎯 Define (Definir)

**Objetivo:** Sintetizar insights y definir el problema

Después de investigar, se analizan los hallazgos para crystalizar un punto de vista claro y definir exactamente qué problema resolver.

### Subcategorías:

| Icono | Subcategoría | Descripción |

## 💻 Develop (Desarrollar)

**Objetivo:** Generar, prototipar y validar soluciones

En esta fase se generan soluciones, se crean prototipos y se testean con usuarios para iterar y mejorar.
