# Insights Cluster (Agrupación de Insights)

## 📖 ¿Qué es?

El **Insights Cluster** (o Agrupación de Insights) es una técnica de síntesis donde se agrupan los insights de investigación en categorías o clusters temáticos para identificar patrones y priorizarlos. Es una de las herramientas más importantes de la fase de Síntesis porque transforma un montaña de datos en conocimiento accionable.

El proceso de clustering te ayuda a ver más allá de los datos individuales y encontrar temas recurrentes. Cuando ves que 5 usuarios diferentes mencionan el mismo problema, ese es un patrón que necesitas abordar.

Esta técnica viene del campo del análisis cualitativo y ha sido adaptada a Design Thinking para organizar findings de user research de manera sistemática.

---

## 🔧 Cómo se usa

### Paso a paso:

1. **Recopila todos los insights** — De entrevistas, encuestas, observación, etc.
2. **Escríbelos en post-its** — Un insight por post-it
3. **Extiende los post-its** — En una superficie grande
4. **Agrupar por temas** — Busca patrones y similitudes
5. **Nombrar los clusters** — Temas que emergen naturalmente
6. **Priorizar** — Por impacto, frecuencia, o alineación con objetivos

### Estructura de un insight:

```
[INSIGHT]
─────────────────────────────────
Qué:        [Observación concreta]
Por qué:    [Interpretación - por qué importa]
Impacto:    [Alto/Medio/Bajo]
Acción:     [Qué hacer con esto]
```

### Consejos:

- **Sé específico** — El insight no es un dato, es una interpretación
- **Incluye la voz del usuario** — Citas que respalden el insight
- **Menos es más** — 5-8 clusters es suficiente
- **Itera** — Los clusters pueden evolucionar
- **Valida con datos** — Asegúrate de que el patrón es real

## 💡 Ejemplo de uso

### Contexto:
Un equipo de e-commerce terminó 20 entrevistas con usuarios. Tienen 87 insights sueltos.

### Proceso de clustering:

**Paso 1: Escribir en post-its**
```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ Insight #1  │  │ Insight #2  │  │ Insight #3  │
│ "No找到el   │  │ "El proceso  │  │ "El checkout │
│ botón de    │  │ de registro  │  │ tiene 5      │
│ checkout"   │  │ toma mucho   │  │ pasos"       │
└──────────────┘  └──────────────┘  └──────────────┘
```

**Paso 2: Agrupar por temas**
```
CLUSTER: FACILIDAD DE USO
├── Los usuarios no encuentran el botón de checkout
├── La navegación es confusa para nuevos usuarios
├── El contraste de colores afecta visibilidad
└── No hay indicador de progreso en el formulario

CLUSTER: CONFIANZA
├── Los usuarios dudan en meter datos de tarjeta
├── Falta información de seguridad visible
├── No hay testimonios o reseñas visibles
└── El precio no incluye impuestos (sorpresa al final)
```

**Paso 3: Priorizar clusters**
| Cluster | Frecuencia | Impacto | Prioridad |
|---------|------------|---------|-----------|
| FACILIDAD DE USO | 8 mentions | Alto | #1 |
| CONFIANZA | 5 mentions | Alto | #2 |
| NAVEGACIÓN | 4 mentions | Medio | #3 |
| PRECIOS | 3 mentions | Medio | #4 |

### Output derivado:
> **Insight prioritario:** Los usuarios de e-commerce necesitan sentir que el proceso es fácil y seguro. El cluster de "Facilidad de Uso" y "Confianza" son los más importantes para abordar.

## 📅 Cuándo usarlo

### Fases recomendadas:

- **Define/Síntesis** — Para organizar findings de investigación
- **Develop** — Para evaluar soluciones contra insights

### Situaciones ideales:

- ✅ Antes de priorizar features
- ✅ Para presentación a stakeholders
- ✅ En workshops de síntesis
- ✅ Cuando tienes muchos datos cualitativos

### No usar cuando:

- ❌ Tienes muy pocos insights (no hay suficiente para agrupar)
- ❌ Los insights son muy heterogéneos (no hay patrones claros)
