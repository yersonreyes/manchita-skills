# Key Facts (Datos Clave)

## 📖 ¿Qué es?

El **Key Facts** es una técnica de síntesis donde se identifican y documentan los hechos más importantes y verificables descubiertos durante la investigación. Son datos concretos, objetivos, que sustentan las decisiones de diseño.

La diferencia clave entre un Key Fact y un insight es que el fact es "lo que pasó" mientras que el insight es "lo que significa". Los facts son el Foundation sobre el cual construyes interpretaciones y decisiones.

Esta técnica es esencial para presentar hallazgos de investigación a stakeholders porque los facts son difíciles de refutar — son datos verificables, no interpretaciones.

---

## 🔧 Cómo se usa

### Las 4 características de un Key Fact:

| Característica | Descripción | Ejemplo |
|----------------|-------------|---------|
| **Verificable** | Se puede demostrar con datos | "70% de usuarios abandona en el paso 3" |
| **Relevante** | Impacta directamente en el problema | Afecta directamente al objetivo |
| **Accionable** | Se puede actuar sobre él | Hay una solución posible |
| **Específico** | No hay ambigüedades | No hay interpretación múltiple |

### Formato para documentar:

```
KEY FACTS
═══════════════════════════════════════════════════

1. [Hecho verificable con dato]
   Fuente: [Entrevista #3, Analytics, Research]
   Implicación: [Qué hacer con este dato]

2. [Otro hecho]
   Fuente: [...]
   Implicación: [...]
```

### Paso a paso:

1. **Revisa toda la investigación** — Entrevistas, encuestas, analytics
2. **Identifica los datos más importantes** — Los que impactan directamente el problema
3. **Verifica cada hecho** — Asegúrate de que son correctos
4. **Documenta con fuente** — Para poder validar después
5. **Deriva implicaciones** — Qué hacer con cada hecho

### Diferencia con Insights:

| Aspecto | Key Facts | Insights |
|---------|-----------|----------|
| Tipo | Datos concretos | Interpretación de datos |
| Pregunta | "¿Qué pasó?" | "¿Por qué pasó y cómo actuar?" |
| Naturaleza | Objetivo | Subjetivo con contexto |
| Ejemplo | "70% abandona checkout" | "Los usuarios abandonan porque el proceso es confuso" |

## 💡 Ejemplo de uso

### Contexto:
Un equipo de e-commerce terminó investigación con usuarios. Necesitan cristalizar los findings para el equipo de producto.

### Key Facts documentados:

```
KEY FACTS - E-COMMERCE CHECKOUT
═══════════════════════════════════════════════════════════

1. El 68% de usuarios que llega al checkout abandona antes de completar
   Fuente: Analytics (data de 3 meses)
   Implicación: Priorizar optimización del checkout

2. El paso que más abandonos tiene es "método de pago" (42% de abandonos)
   Fuente: Heatmaps + Analytics
   Implicación: Simplificar o mostrar más opciones de pago

3. El 80% de usuarios accede desde móvil
   Fuente: Device analytics
   Implicación: Mobile-first en el diseño del checkout

4. Los usuarios que usan "pagar con un click" tienen 3x más probabilidad de completar
   Fuente: A/B test anterior
   Implicación: Priorizar feature de checkout rápido

5. La principal razón de abandono en el paso de shipping es "costo muy alto" (56%)
   Fuente: Encuesta de abandono (n=500)
   Implicación: Considerar envío gratis a partir de cierto monto

6. El tiempo promedio para completar checkout es 4.2 minutos (desktop), 6.8 minutos (móvil)
   Fuente: Session recordings
   Implicación: Simplificar flujos, especialmente en móvil
```

### Uso derivado:

| stakeholder | Cómo se usa |
|-------------|--------------|
| Producto | Priorizar roadmap de mejoras de checkout |
| Diseño | Enfocar en mobile y simplificar flujos |
| Marketing | Diseñar promoción de shipping gratis |
| Finance | Modelar el costo de envío gratis |

## 📅 Cuándo usarlo

### Fases recomendadas:

- **Define/Síntesis** — Para cristalizar research
- **Develop** — Para validar decisiones de diseño
- **Deliver** — Para presentar a stakeholders

### Situaciones ideales:

- ✅ Para presentar a stakeholders
- ✅ Para validar decisiones
- ✅ Como fundamento de insights
- ✅ Para alineación de equipo

### No usar cuando:

- ❌ Los datos no son verificables
- ❌ Son interpretaciones, no facts
