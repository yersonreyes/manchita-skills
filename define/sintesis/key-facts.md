# Key Facts

## ¿Qué es?

Key Facts es una técnica de síntesis donde se identifican y documentan los hechos más importantes y verificables descubiertos durante la investigación. Son datos concretos que sustentan decisiones de diseño.

## Cómo se usa

### Características de un Key Fact:

1. **Verificable** — Se puede demostrar con datos
2. **Relevante** — Impacta directamente en el problema
3. **Accionable** — Se puede actuar sobre él
4. **Específico** — No ambigüedades

### Formato:

```
KEY FACTS

1. [Hecho verificable con dato]
   Fuente: [Entrevista #3, Analytics, etc.]
   
2. [Otro hecho]
   Fuente: [...]
```

### Ejemplos:

| Key Fact | Fuente |
|----------|--------|
| 70% de usuarios abandona el checkout en paso 3 | Analytics |
| El tiempo promedio de espera es 15 min | Entrevistas |
| 80% de usuarios usa móvil | Device analytics |

## Diferencia con Insights:

| Key Facts | Insights |
|-----------|----------|
| Datos concretos | Interpretación de datos |
| "Qué" | "Por qué" y "cómo actuar" |
| Objetivo | Subjetivo con contexto |

## Cuándo usarlo

- En **Define** para cristalizar research
- Para presentar a stakeholders
- Para validar decisiones
- Como fundamento de insights
