# POV (Point of View)

## ¿Qué es?

El Point of View (POV) es una declaración concisa que define el problema de diseño desde la perspectiva del usuario, combinando quién es, cuál es su necesidad y por qué es importante. Es el fundamento para el How Might We.

## Cómo se usa

### Template de POV:

```
[Usuario específico] necesita [necesidad/insight] 
porque [insight sorprendente].
```

### Componentes:

1. **Usuario** — Quién es (basado en research)
2. **Necesidad** — Qué necesita o quiere
3. **Insight** — Por qué es importante (insight de research)

### Ejemplos:

| Usuario | Necesidad | Insight | POV |
|---------|-----------|---------|-----|
| Juan, padre ocupado | Completar compras rápido | No tiene tiempo | Juan, padre ocupado, necesita completar compras rápido porque no tiene tiempo para procesos largos |
| María, emprendedora | Facturar fácil | Pierde clientes | María, emprendedora, necesita facturar fácil porque pierde clientes por no poder emitir facturas rápidamente |

### Proceso para crear POV:

1. **Tener personas** — Definir usuario target
2. **Tener insights** — De investigación
3. **Combinar** — En el template
4. **Iterar** — Refinar hasta que sea claro y accionable

## Cuándo usarlo

- En **Define** para crystalizar el problema
- Antes del How Might We
- Para alinear al equipo
- Como referencia durante el diseño
