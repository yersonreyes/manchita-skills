# Metáfora del Problema (Problem Metaphor)

## 📖 ¿Qué es?

La **Metáfora del Problema** es una técnica de síntesis que representa el problema de diseño usando una metáfora o analogía. Ayudan a entender el problema desde otra perspectiva y facilita la comunicación con stakeholders que pueden no entender los detalles técnicos.

La magia de las metáforas es que activan el pensamiento lateral — al comparar el problema con algo diferente, revelan aspectos que no se habían visto. Además, las metáforas hacen que los problemas sean más memorables y comprensibles.

Esta técnica viene del pensamiento de diseño y la comunicación estratégica, donde las metáforas son usadas para hacer compleja información más digerible.

---

## 🔧 Cómo se usa

### Paso a paso:

1. **Define el problema** — Claro y específico
2. **Busca metáforas** — ¿Qué se parece a esto en otro contexto?
3. **Explora la metáfora** — ¿Qué insights te da? ¿Qué sugiere?
4. **Aplica insights** — Vuelve al problema original con nueva perspectiva

### Tipos de metáforas:

| Tipo | Ejemplo | Cuándo usarla |
|------|---------|---------------|
| **Del mundo físico** | "Como un río con obstáculos" | Problemas de flujo |
| **De la naturaleza** | "Como una planta que necesita luz" | Problemas de crecimiento |
| **De la vida cotidiana** | "Como organizar un closet" | Problemas de organización |
| **De otros dominios** | "Como un aeropuerto" | Problemas de sistema |

### Ejemplos de metáforas:

| Problema | Metáfora | Nuevos insights |
|----------|----------|-----------------|
| Usuarios no completan onboarding | "Es como perder las llaves cada vez que salimos" | Necesitamos "encontrar las llaves" antes de salir |
| Sistema lento | "Es como esperar en una cola infinita" | La gente necesita ver progreso |
| Datos dispersos | "Es como tener un cuarto donde todo está tirado" | Necesitamos "un lugar para cada cosa" |
| Users no encuentran feature | "Es como buscar una aguja en un pajar" | Necesitamos hacer la aguja más visible |

### Beneficios de usar metáforas:

- **Comunicación** — Más fácil de explicar a no-diseñadores
- **Perspective** — Ver el problema desde otro ángulo
- **Creatividad** — La metáfora sugiere soluciones
- **Empatía** — Conecta emocionalmente con el problema
- **Memorabilidad** — Se queda en la mente

### Consejos:

- **No forzar** — La metáfora debe sentirse natural
- **Que sea compartida** — Que el equipo la entienda
- **Que sea fértil** — Que sugiera varias direcciones
- **Revisa después** — ¿La metáfora sigue haciendo sentido?

## 💡 Ejemplo de uso

### Contexto:
Un equipo está trabajando en un problema donde los usuarios no pueden encontrar la información que buscan en la app de soporte.

### Problema original:
> Los usuarios no encuentran respuestas a sus preguntas en la app de soporte y terminan contactingando soporte humano.

### Exploración de metáforas:

**Metáfora 1: "Como un laberinto"**
- El usuario entra pero no sabe cómo salir
- Está lleno de giros y callejones sin salida
- Insights: Necesitamos "salidas claras", "señalización"

**Metáfora 2: "Como una biblioteca sin índice"**
- Hay muchísima información pero no se puede encontrar
- Tienes que leer todo para encontrar lo que buscas
- Insights: Necesitamos "categorización", "búsqueda inteligente"

**Metáfora 3: "Como pedir direcciones a un extraño"**
- No sabes quién puede ayudarte
- Las personas que preguntas no tienen la respuesta
- Insights: Necesitamos "expertos disponibles", "self-service claro"

### Metáfora seleccionada:
> "La app de soporte es como una biblioteca sin índice"

### Insights derivados:
- Agregar búsqueda con inteligencia artificial
- Organizar contenido por tipo de pregunta
- Mostrar "artículos populares" al inicio
- Agregar "preguntas relacionadas" al final de cada artículo

### Uso:
> **En presentación a stakeholders:** "Imaginen una biblioteca donde tienes que leer todos los libros para encontrar uno. Eso es lo que sienten nuestros usuarios. Ahora, ¿cómo creamos el índice?"

## 📅 Cuándo usarlo

### Fases recomendadas:

- **Define** — Para crystalizar el problema
- **Develop** — Para generar ideas durante ideación

### Situaciones ideales:

- ✅ En presentaciones a stakeholders
- ✅ Para generar ideas durante ideación
- ✅ Para hacer memorable el problema
- ✅ Cuando necesitas comunicar a no-diseñadores

### No usar cuando:

- ❌ El problema es muy técnico (la metáfora puede confundir)
- ❌ Los stakeholders prefieren datos a metáforas
