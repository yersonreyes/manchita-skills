# Stakeholder Map (Síntesis)

## 📖 ¿Qué es?

El **Stakeholder Map** en la fase de Síntesis es una herramienta visual para representar las relaciones, intereses y poder de influencia de todos los actores relacionados con un proyecto o producto. A diferencia del Stakeholder Map de Discover (que es más exploratorio), aquí se enfoca en el análisis y la estrategia de comunicación.

En Síntesis, ya conoces el ecosistema y las relaciones entre actores. El mapa se convierte en una herramienta de gestión para planificar cómo te vas a comunicar, influir, y coordinar con cada stakeholder durante el proyecto.

Esta herramienta es crítica para el éxito del proyecto porque los stakeholders pueden ser tanto facilitadores como bloqueadores. Un buen mapeo te permite gestionar relaciones proactivamente.

## 🔧 Cómo se usa

### La Matriz de Poder/Interés:

```
                         INTERÉS
                    ┌───────────┬───────────┐
                    │  BAJO     │   ALTO    │
        ┌───────────┼───────────┼───────────┤
    A   │  KEEP     │  MANAGE   │           │
    L   │  SATISFIED│  CLOSELY  │           │
    T   │           │           │           │
    O   ├───────────┼───────────┼───────────┤
        │   BAJO    │           │           │
        │  INTEREST │  MONITOR  │   KEEP    │
        │           │           │  INFORMED │
        └───────────┴───────────┴───────────┘
                    INFLUENCIA
```

### Las 4 categorías:

| Categoría | Stakeholders | Estrategia |
|-----------|-------------|------------|
| **Manage Closely** | Alta influencia + Alto interés | Involucrar en decisiones, mantener informados, buscar input |
| **Keep Satisfied** | Alta influencia + Bajo interés | Actualizar regularmente, evitar sorpresas, mantener satisfechos |
| **Keep Informed** | Baja influencia + Alto interés | Consultar, compartir información, reconocer contribuciones |
| **Monitor** | Baja influencia + Bajo interés | Observar, mínimo esfuerzo de comunicación |

### Tipos de relaciones a mapear:

| Tipo | Qué incluir |
|------|-------------|
| **De influencia** | Quién puede afectar el proyecto (positiva o negativamente) |
| **De dependencia** | Quién necesita qué (recursos, decisiones, información) |
| **De comunicación** | Cómo y con qué frecuencia te comunicas |
| **De decisión** | Quién tiene poder de veto o aprobación |

### Paso a paso:

1. **Identifica stakeholders** — Todos los actores relevantes
2. **Evalúa su poder e interés** — Para cada uno, dónde cae en la matriz
3. **Mapea relaciones** — Cómo se relacionan entre sí
4. **Define estrategias** — Para cada cuadrante
5. **Planifica comunicación** — Frecuencia, canal, contenido

## 💡 Ejemplo de uso

### Contexto:
Una empresa de retail va a lanzar un nuevo e-commerce. El equipo hace el Stakeholder Map para planificar el proyecto.

### Stakeholder Map creado:

```
                         INTERÉS
                    ┌───────────┬───────────┐
                    │  BAJO     │   ALTO    │
        ┌───────────┼───────────┼───────────┤
    A   │  KEEP     │  MANAGE   │           │
    L   │  SATISFIED│  CLOSELY  │           │
    T   │           │           │           │
    O   │  legal    │  equipo   │  equipo   │
    │   │  IT       │  producto │  diseño   │
    ├───────────┼───────────┼───────────┤
    │   BAJO    │           │   KEEP    │
    │  INTEREST │  MONITOR  │  INFORMED  │
    │           │           │           │
    │  medios   │  soporte   │  clientes │
    │          │  técnico   │  final    │
    └───────────┴───────────┴───────────┘
                    INFLUENCIA
```

### Estrategias derivadas:

| Stakeholder | Categoría | Estrategia |
|-------------|-----------|------------|
| VP Producto | Manage Closely | Weekly sync, involved in decisions |
| Equipo Diseño | Manage Closely | Daily standups, collaboration |
| Legal | Keep Satisfied | Monthly updates, avoid surprises |
| IT | Keep Satisfied | Bi-weekly syncs, clear requirements |
| Clientes final | Keep Informed | User testing sessions, feedback |
| Medios | Monitor | General news, press releases only |

### Acción tomada:
> "Prioritamos alinear con Legal al inicio para evitar bloqueo al final del proyecto. Les dimos un timeline claro y los mantenemos actualizados cada mes."

## 📅 Cuándo usarlo

### Fases recomendadas:

- **Define/Síntesis** — Para planificar comunicación
- **Develop** — Para validar que las soluciones consideran stakeholders

### Situaciones ideales:

- ✅ Antes de presentar a clientes/stakeholders
- ✅ Para identificar bloqueos potenciales
- ✅ En gestión de proyectos complejos
- ✅ Cuando hay múltiples dependencias internas/externas

### No usar cuando:

- ❌ El proyecto es muy simple y tiene pocos stakeholders
- ❌ No tienes información sobre los stakeholders (investiga primero)
