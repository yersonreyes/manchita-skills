# User Persona (Síntesis)

## 📖 ¿Qué es?

La **User Persona** en la fase de Síntesis es una representación arquetípica de un usuario objetivo, basada en datos reales de investigación. A diferencia de la Persona de Discover (que es más exploratoria), aquí la Persona está completamente validada y refinada, lista para guiar todas las decisiones de diseño.

La Persona de Síntesis es más completa y detallada porque integra toda la información collected durante la investigación. Es el "perfil final" del target del proyecto.

La Persona es una de las herramientas más poderosas de Design Thinking porque crea un "cliente imaginario" que el equipo puede usar para tomar decisiones. Cuando hay un desacuerdo, la pregunta "¿Qué haría la Persona?" usually resolves it.

## 🔧 Cómo se usa

### Componentes de la Persona:

| Sección | Qué incluir |
|---------|-------------|
| **Nombre y foto** | Nombre memorable, foto que represente al usuario |
| **Demográficos** | Edad, trabajo, ingresos, ubicación, familia |
| **Biografía** | Contexto personal y profesional |
| **Metas y motivaciones** | Qué quiere lograr, por qué le importa |
| **Frustraciones** | Qué le frustra, qué barreras enfrenta |
| **Comportamiento** | Cómo usa tecnología, canales preferidos |
| **Citación** | Quote real del usuario que captura su esencia |

### Fuentes para crear personas:

- Entrevistas cualitativas
- Encuestas cuantitativas
- Datos de analytics
- Observación (safari, shadowing)
- Research de competidores

### Paso a paso:

1. **Reune datos** — De toda la investigación
2. **Identifica patrones** — Busca segmentos naturales
3. **Construye el perfil** — Llena cada sección
4. **Valida** — Con el equipo
5. **Úsala** — En todas las decisiones de diseño

### Consejos:

- **Usa datos reales** — No specules
- **Sé específico** — "María, 32, diseñadora" > "usuario millennial"
- **Incluye una cita real** — De las entrevistas
- **No más de 2-3 por proyecto** — Demasiadas = parálisis
- **Que sea recordada** — Ponle un nombre que el equipo use

## 💡 Ejemplo de uso

### Contexto:
Una fintech ha terminado su investigación con freelancers. Crean las Personas finales.

### Persona creada:

```
┌─────────────────────────────────────────────┐
│         MARÍA, LA FREELANCER MINIE         │
│   [Foto: Mujer de 30s, looks profesional   │
│   pero accesible, working from coffee shop] │
├─────────────────────────────────────────────┤
│  DEMOGRAFICS                                │
│  • Edad: 32 años                           │
│  • Trabajo: Diseñadora freelancer          │
│  • Ingresos: $25k-40k MXN/mes              │
│  • Ubicación: Ciudad de México             │
│  • Familia: Soltera, vive con roommate     │
├─────────────────────────────────────────────┤
│  BIO                                        │
│  María trabaja como diseñadora freelance   │
│  desde hace 4 años. Le gusta su flexibilidad│
│  pero le cuesta organizar sus finanzas.    │
│  Trabaja desde casa y coffee shops.        │
│  Pasa mucho tiempo en Instagram y TikTok. │
├─────────────────────────────────────────────┤
│  METAS & MOTIVACIONES                       │
│  • Sentir que sus finanzas están organizadas│
│  • Tener un fondo de emergencia            │
│  • No depender de un solo cliente          │
│  • Encontrar clientes de forma consistente  │
├─────────────────────────────────────────────┤
│  FRUSTRATIONS                               │
│  • No entender de finanzas                 │
│  • Procesos complicados                    │
│  • Pierde tiempo en administración         │
│  • Le da miedo invertir                    │
├─────────────────────────────────────────────┤
│  COMPORTAMIENTO                             │
│  • Heavy user de Instagram/TikTok para      │
│    aprender                                │
│  • Usa su teléfono para todo              │
│  • Prefiere videos cortos                   │
│  • Confía en recomendaciones de amigos    │
├─────────────────────────────────────────────┤
│  QUOTE                                      │
│  "Sé que debería invertir, pero me da     │
│  miedo perder lo poco que tengo. Necesito  │
│  algo que me explique en palabras simples  │
│  qué está pasando con mi dinero."           │
└─────────────────────────────────────────────┘
```

### Uso en decisiones:

| Decisión | Pregunta desde la persona | Respuesta |
|----------|--------------------------|-----------|
| Lenguaje | "¿María entendería?" | → Simple, sin jerga |
| Canales | "¿María dónde está?" | → Instagram, TikTok |
| Onboarding | "¿Tiene tiempo María?" | → Corto, incremental |
| Feature | "¿María lo necesitaría?" | → Focus en lo básico primero |

## 📅 Cuándo usarlo

### Fases recomendadas:

- **Define/Síntesis** — Para crystalizar el target
- **Develop** — Para guiar decisiones de diseño
- **Deliver** — Para validar con usuarios target

### Situaciones ideales:

- ✅ Para guiar todas las decisiones de diseño
- ✅ Para comunicar insights a stakeholders
- ✅ Para resolver debates en el equipo
- ✅ En todas las fases subsecuentes

### No usar cuando:

- ❌ No tenés datos de investigación
- ❌ Tenés audiencias muy diferentes (usa múltiples personas)
