# Mapa de Empatía (Síntesis)

## 📖 ¿Qué es?

El **Mapa de Empatía** en la fase de Síntesis es una herramienta visual que captura las perspectivas y emociones de un usuario específico. A diferencia del Mapa de Empatía de Discover (creado durante la investigación), el Mapa de Empatía de Síntesis se crea/refina después de completar la investigación para representar al usuario objetivo final.

La diferencia clave es que en Síntesis el mapa está más pulido, validado con datos reales, y se usa como input directo para el POV y las decisiones de diseño. Es la versión "final" que representa al target del proyecto.

Esta herramienta te permite profundizar en el usuario más allá de los datos demográficos, capturando su mundo emocional, sus pensamientos, y sus acciones en un formato visual que el equipo puede entender rápidamente.

## 🔧 Cómo se usa

### Estructura:

```
┌──────────────────────────────────────────────────────────────────┐
│                          USUARIO: [Nombre]                      │
├────────────────────┬────────────────────────────────────────────┤
│     VE             │            OYE                              │
│  Qué ve en         │  Qué escucha el                            │
│  su entorno?       │  usuario?                                 │
│                    │                                            │
├────────────────────┼────────────────────────────────────────────┤
│     PIENSA         │            SIENTE                          │
│  Qué piensa el    │  Qué emociones                            │
│  usuario?         │  experimenta?                             │
│  (miedos,         │  (frustración,                             │
│   creencias)      │   motivación, alegría)                      │
├────────────────────┼────────────────────────────────────────────┤
│     DICE           │            HACE                             │
│  Qué dice el      │  Qué hace el                              │
│  usuario?         │  usuario?                                 │
│                    │                                            │
└────────────────────┴────────────────────────────────────────────┘
```

### Paso a paso:

1. **Elegir una persona** — De las creadas en investigación
2. **Completar cada cuadrante** — Con findings de research específicos
3. **Validar** — Asegurar que sea real y verificable
4. **Compartir** — Con el equipo y stakeholders

### Los 6 cuadrantes:

| Cuadrante | Qué incluir | Ejemplo |
|-----------|-------------|---------|
| **VE** | Qué ve en su entorno | "Ve que sus amigos usan apps de inversión" |
| **OYE** | Qué escucha de otros | "Escucha que 'invertir es muy complicado'" |
| **PIENSA** | Qué pasa por su mente | "Piensa que no tiene suficiente dinero" |
| **SIENTE** | Qué emociones tiene | "Siente frustración cuando ve sus ahorros" |
| **DICE** | qué dice en voz alta | "Dice 'yo no entiendo de finanzas'" |
| **HACE** | Cómo actúa | "Abre la app de su banco pero no invierte" |

### Consejos:

- **Usa datos de investigación** — No specules
- **Diferencia lo que dice vs hace** — A menudo son diferentes
- **Sé específico** — Evita generalidades
- **Incluye la voz del usuario** — Citas reales

## 💡 Ejemplo de uso

### Contexto:
El equipo de una fintech ha terminado su investigación con freelancers. Ahora crean el Mapa de Empatía final para el POV.

### Mapa de Empatía de Síntesis:

```
┌──────────────────────────────────────────────────────────────────┐
│                    USUARIO: María, Freelancer                   │
├────────────────────┬────────────────────────────────────────────┤
│     VE             │            OYE                              │
│ • Ve en redes       │ • Escucha de otros                          │
│   sociales a        │   freelancers que "es                     │
│   influencers       │   un dolor la                               │
│   promoviendo       │   facturación"                             │
│   inversiones       │ • Escucha a su                               │
│ • Ve apps de        │   mamá que dice "no                        │
│   banking           │   gastes en                                 │
│   tradicionales     │   tonterías"                                │
├────────────────────┼────────────────────────────────────────────┤
│     PIENSA         │            SIENTE                           │
│ • "No tengo         │ • Siente                                  │
│   suficiente        │   FRUSTRACIÓN                              │
│   dinero para       │   cuando no sabe                            │
│   invertir"         │   dónde están sus                          │
│ • "Invertir es      │   ganancias                                 │
│   para gente        │ • Siente                                │
│   con dinero"       │   ESPERANZA                                │
│ • "No entiendo      │   cuando ve                               │
│   cómo funcionan    │   opciones de                              │
│   las cosas"         │   inversión simples                        │
├────────────────────┼────────────────────────────────────────────┤
│     DICE           │            HACE                             │
│ • "Yo no entiendo │ • Abre la app                               │
│   de finanzas"    │   3+ veces al día                          │
│ • "Me da miedo   │ • Guarda links de                            │
│   perder mi        │   tutoriales que                            │
│   dinero"          │   nunca ve                                  │
│ • "Solo quiero   │ • Pide consejos                              │
│   algo seguro"    │   a amigos                                  │
└────────────────────┴────────────────────────────────────────────┘
```

### Uso derivado:
> Este mapa informará directamente el POV:
> "María, freelancer que necesita invertir sus ganancias porque actualmente no tiene claro dónde está su dinero y siente miedo de tomar decisiones financieras."

## 📅 Cuándo usarlo

### Fases recomendadas:

- **Define/Síntesis** — Para profundizar en el usuario target
- **Develop** — Para validar que las soluciones resuenan con el usuario

### Situaciones ideales:

- ✅ Para validar assumptions con el equipo
- ✅ En workshops de empathy antes de ideación
- ✅ Antes de crear el POV
- ✅ Para comunicar insights a stakeholders

### No usar cuando:

- ❌ No tenés datos de investigación (no es para specular)
- ❌ El equipo ya tiene clara a la audiencia (puede ser redundante)
