# Brief

## ¿Qué es?

El Brief es un documento estratégico que sintetiza toda la información del proyecto de diseño: objetivos, scope, restricciones,Timeline, stakeholders y entregables. Es el punto de referencia para todo el proyecto.

### Tips para un buen Brief:

- **Ser específico** — Evitar vaguedades
- **Ser realista** — Considerar constraints
- **Ser conciso** — 1-2 páginas máximo
- **Ser accionable** — Que guíe decisiones
- **Ser compartido** — Todos deben conocerlo

## Cuándo usarlo

- En **Define** al inicio del proyecto
- Para alinear equipo y stakeholders
- Como referencia durante el proyecto
- Para evaluarscope creep
