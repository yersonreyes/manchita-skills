# Desafío de Diseño (Design Challenge)

## ¿Qué es?

El Desafío de Diseño es una declaración que articula claramente el problema o reto que el equipo de diseño debe resolver. Define el scope, las restricciones y el objetivo del proyecto de diseño.

## Cómo se usa

### Template:

```
Cómo podemos [acción específica] 
para [usuario/beneficiario] 
dentro de [contexto/restricciones] 
para [beneficio/resultado esperado]?
```

### Elementos del Design Challenge:

1. **Qué resolver** — El problema específico
2. **Para quién** — El usuario o beneficiary
3. **En qué contexto** — Restricciones, limitaciones
4. **Qué resultado** — Beneficio esperado

### Ejemplos:

| Problema | Design Challenge |
|----------|------------------|
| Usuarios abandonan checkout | "¿Cómo podemos reducir el tiempo de checkout para usuarios móviles, para que completen su compra sin abandonar?" |
| Empleados no usan el sistema | "¿Cómo podemos hacer que los empleados de bodega usen el sistema de inventario, para que los datos sean precisos en tiempo real?" |

### Proceso:

1. **Definir el problema** — Basado en research
2. **Identificar constraints** — Tiempo, budget, tech
3. **Articular el reto** — Usar el template
4. **Validar** — Con stakeholders

## Cuándo usarlo

- En **Define** para establecer el scope
- Al inicio de un proyecto de diseño
- Para alinear expectativas con clientes
- Para guiar la ideación
