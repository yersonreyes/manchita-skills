# El Proceso de Diseño

## Resumen

El proceso de diseño es un marco metodológico que guía a los equipos de diseño desde la identificación de un problema hasta la entrega de una solución viable. Este proceso no es lineal ni único, sino adaptativo y iterativo.

Una de las metodologías más utilizadas es el **Double Diamond** (Doble Diamante), propuesta por el British Design Council.

---

## 🎨 El Modelo Double Diamond

El Double Diamond divide el proceso de diseño en cuatro fases, alternando entre divergencia y convergencia:


## Fases del Proceso

### 🔍 Fase 1: Discover (Descubrir)

**Objetivo:** Entender el problema y el contexto

Esta fase consiste en investigar y observar para comprender deeply el problema antes de buscar soluciones.

**Actividades típicas:**
- Entrevistas con usuarios
- Observación contextual
- Benchmarking
- Mapas de empatía
- Investigación de mercado

**Preguntas guía:**
- ¿Qué está pasando?
- ¿Por qué ocurre?
- ¿Quiénes están afectados?

---

### 🎯 Fase 2: Define (Definir)

**Objetivo:** Sintetizar los insights y definir el problema a resolver

Después de investigar, es hora de analizar los hallazgos y crystalizar un punto de vista claro.

**Actividades típicas:**
- Affinity mapping
- User personas
- Jobs-to-be-done
- Problem statements
- How Might We (HMW)

**Preguntas guía:**
- ¿Qué problema específico resuelves?
- ¿Para quién?
- ¿Por qué importa?

---

### 💻 Fase 3: Develop (Desarrollar)

**Objetivo:** Generar y prototipar soluciones

Aquí es donde la creatividad explode. Se generan múltiples ideas y se prototipan para testing.

**Actividades típicas:**
- Brainstorming
- Sketching
- Wireframing
- Prototipado
- Testing con usuarios

**Preguntas guía:**
- ¿Qué soluciones posibles existen?
- ¿Cómo podemos resolver el problema?

---

### 🚀 Fase 4: Deliver (Entregar)

**Objetivo:** Validar, implementar y lanzar

La solución final se refina, implementa y lanza al mundo real.

**Actividades típicas:**
- Pruebas finales
- Documentación
- Lanzamiento
- Iteración post-lanzamiento
-handoff a desarrollo

**Preguntas guía:**
- ¿La solución funciona?
- ¿Está lista para producción?

---

## Iteración

El proceso de diseño es **iterativo**, no lineal. Es común volver a fases anteriores:

- Feedback de users en Develop → volver a Discover
- Insights en Deliver → nuevo ciclo de diseño

> "Design is a loop, not a line."
