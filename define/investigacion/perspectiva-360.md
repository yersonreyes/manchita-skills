# Perspectiva 360

## ¿Qué es?

La Perspectiva 360 es una técnica de investigación que busca entender un tema, producto o problema desde todos los ángulos posibles: usuarios, stakeholders, competencia, tecnología, negocio, etc. Es una visión holística.

## Cómo se usa

### Dimensiones típicas:

1. **Usuario** — Need, behaviors, pain points
2. **Negocio** — Revenue, costs, KPIs
3. **Tecnología** — Feasibility, constraints
4. **Competencia** — Alternatives, gaps
5. **Stakeholders** — Internal, partners
6. **Legal/Regulatorio** — Compliance
7. **Trend/Industry** — Future, disruption

### Proceso:

```
1. Definir el objeto de estudio
2. Identificar todas las perspectivas
3. Investigar cada una
4. Consolidar insights
5. Crear resumen 360
```

### Matriz de Perspectiva 360:

| Perspectiva | Pregunta Clave | Fuente |
|-------------|----------------|--------|
| Usuario | ¿Qué necesita? | Entrevistas, encuestas |
| Negocio | ¿Cómo genera valor? | Análisis financiero |
| Tecnológico | ¿Es técnicamente posible? | Tech assessment |
| Competencia | ¿Qué alternatives hay? | Benchmarking |
| Stakeholder | ¿Quién tiene poder? | Stakeholder map |
| Legal | ¿Qué restricciones hay? | Abogado/compliance |
| Tendencias | ¿Hacia dónde va? | Research de mercado |

## Cuándo usarlo

- En **Define** para visión completa del problema
- Antes de definición de producto
- Para validar viabilidad
- En strategic planning
